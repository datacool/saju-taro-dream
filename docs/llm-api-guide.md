# LLM 모델 호출 기능 설명서

> 작성일: 2026-03-31
> 대상 모델: `gpt-4o` (OpenAI)
> 인증: `.env.local` → `OPENAI_API_KEY`

---

## 개요

이 서비스는 **OpenAI `gpt-4o`** 를 3개 기능에서 호출합니다.
사주 계산·카드 뽑기·입력 수집 등 **로직은 코드에서 직접 처리**하고,
그 결과를 LLM에게 전달하여 **페르소나 기반 해석 텍스트만** 생성하게 합니다.

모든 호출은 **스트리밍 방식** (`stream: true`) 으로 동작하여 글자가 실시간으로 화면에 출력됩니다.

---

## 1. 사주 분석 — `/api/saju`

### 호출 시점
사용자가 생년월일·시간·이름·성별을 입력하고 **"사주 분석"** 또는 **"오늘의 운세"** 버튼을 누를 때.

### LLM 이전에 코드가 처리하는 것
| 처리 항목 | 담당 모듈 |
|-----------|-----------|
| 만세력 기반 사주팔자(년·월·일·시주) 계산 | `src/lib/saju.ts` → `calculateSaju()` |
| 천간·지지 한자 및 오행 변환 | `src/lib/saju.ts` |
| 오행 분포(목·화·토·금·수 개수) 집계 | `src/lib/saju.ts` |
| 오행 레이더 차트·바 그래프 렌더링 | `src/app/result/page.tsx` |

### LLM에 전달하는 프롬프트 (user 메시지)
```
이름: 홍길동 (남성)
생년: 甲子년 / 띠: 쥐띠

[사주팔자]
년주: 甲子(갑자) - 목+수
월주: 丙寅(병인) - 화+목
일주: 戊午(무오) - 토+화
시주: 庚申(경신) - 금+금

[오행 분포] 목(2) 화(2) 토(1) 금(2) 수(1)
일간(나 자신): 戊(무) - 토의 기운
```

### System 프롬프트 (페르소나)
**박진인(朴眞人)** — 조선 시대 사주 명리학 도인
- 고어 종결형 100% 사용: `~하오`, `~이오`, `~하리라`, `~하도다`
- `~입니다`, `~합니다` 등 현대 경어체 절대 금지
- 1인칭 `이 박진인`, 2인칭 `자네`
- `허허`, `오호라` 등 감탄사 삽입
- 한자어 최초 등장 시 뜻 병기: `일간(日干, 나 자신을 나타내는 글자)`

### LLM이 생성하는 섹션

**standard 모드 (풀 사주 분석):**
```
## 타고난 기질과 성격
## 직업운과 커리어
## 연애운과 배우자
## 재물운
## 건강운
## 2026년 운세 총평
```

**daily 모드 (오늘의 운세):**
```
## 오늘의 종합운
## 오늘의 애정운
## 오늘의 금전운
## 오늘의 건강운
## 오늘의 행운 포인트
```

### 호출 파라미터
| 파라미터 | 값 |
|----------|----|
| model | `gpt-4o` |
| max_tokens | `2000` |
| temperature | `0.75` |
| stream | `true` |

## 2. 타로 리딩 — `/api/tarot`

### 호출 시점
사용자가 질문을 입력하고 카드 3장을 선택한 뒤 **"해석하기"** 버튼을 누를 때.

### LLM 이전에 코드가 처리하는 것
| 처리 항목 | 담당 모듈 |
|-----------|-----------|
| 78장 타로 전체 데이터 정의 | `src/lib/tarot.ts` |
| 중복 없이 카드 N장 뽑기 (셔플) | `src/lib/tarot.ts` → `drawCards()` |
| 역방향(역위) 여부 결정 (25% 확률) | `src/app/tarot/page.tsx` |
| 카드 플립 애니메이션 | `src/app/tarot/page.tsx` |
| 카드 도감(본 카드 기록) 저장 | `src/lib/gamification.ts` → `saveSeenCards()` |

### LLM에 전달하는 프롬프트 (user 메시지)
```
[질문]
지금 이직을 고려하고 있는데 어떻게 해야 할까요?

[뽑힌 카드 - 3카드 스프레드]
1. 상황 카드: 마법사 (The Magician) — 완드 수트, 정방향
   키워드: 의지, 창조, 집중
2. 조언 카드: 달 (The Moon) — 메이저 아르카나, 역방향
   키워드: 혼란, 불안, 숨겨진 진실
3. 결과 카드: 별 (The Star) — 메이저 아르카나, 정방향
   키워드: 희망, 치유, 영감
```

### System 프롬프트 (페르소나)
**마담 셀레스트(Madame Céleste)** — 19세기 파리 출신 타로 마스터
- 시적이고 신비로운 언어 + 실질적 통찰 병행
- 가끔 프랑스어 한 구절 삽입: `"Les étoiles disent..."` (별들이 말합니다)
- 부정적 카드도 성장·변화 관점에서 재해석
- 스토리텔링 방식으로 카드 연결

### LLM이 생성하는 섹션
```
## 전체적인 메시지
## 상황 카드: [카드명] 해석
## 조언 카드: [카드명] 해석
## 결과 카드: [카드명] 해석
## 종합 조언과 앞으로의 방향
```

### 호출 파라미터
| 파라미터 | 값 |
|----------|----|
| model | `gpt-4o` |
| max_tokens | `1800` |
| temperature | `0.8` |
| stream | `true` |

## 3. 꿈해몽 — `/api/dream`

### 호출 시점
사용자가 꿈 내용을 입력하고 감정·테마를 선택한 뒤 **"꿈 해석하기"** 버튼을 누를 때.

### LLM 이전에 코드가 처리하는 것
| 처리 항목 | 담당 모듈 |
|-----------|-----------|
| 감정 선택지 8종 제공 | `src/app/dream/page.tsx` |
| 테마 선택지 12종 제공 | `src/app/dream/page.tsx` |
| 꿈 일기 localStorage 저장/조회 | `src/app/dream/page.tsx` |

### LLM에 전달하는 프롬프트 (user 메시지)
```
[꿈 내용]
하늘을 날다가 갑자기 땅으로 떨어졌는데 다치지 않았습니다.
밝은 빛이 보이고 누군가가 손을 잡아줬어요.

꿈 속 감정: 두려움, 설렘
주요 테마: 비행, 추락, 빛
```

### System 프롬프트 (페르소나)
**몽신(夢神) 아라** — 수천 년간 인간의 꿈을 기록한 신령
- 부드럽고 몽환적 말투: `"당신의 꿈이 내게 흘러왔어요"`, `"꿈의 결이 느껴져요"`
- 칼 융(Carl Jung) 분석심리학 + 동양 전통 해몽 결합
- 불길한 예언 금지, 성장·자기이해 중심 해석

### LLM 응답의 특수 포맷 — 꿈 유형 자동 분류
응답 첫 줄에 반드시 아래 형식으로 꿈 유형을 출력하도록 지시:
```
__TYPE__길몽__
```
프론트엔드(`src/app/dream/page.tsx`)가 이 태그를 파싱하여 화면 상단 배지로 표시함.
6가지 유형: `길몽` / `예지몽` / `정화몽` / `불안몽` / `흉몽` / `일상몽`

### LLM이 생성하는 섹션
```
## 꿈의 전체적 메시지
## 주요 상징 해석
## 심리학적 관점
## 운세적 의미
## 앞으로의 조언
```

### 호출 파라미터
| 파라미터 | 값 |
|----------|----|
| model | `gpt-4o` |
| max_tokens | `1800` |
| temperature | `0.8` |
| stream | `true` |

## 전체 흐름 요약

```
사용자 입력
    │
    ▼
[코드] 사주 계산 / 카드 셔플·뽑기 / 입력 정제
    │   (OPENAI_API_KEY 불필요)
    ▼
POST /api/{saju|tarot|dream}
    │
    ▼
[OpenAI gpt-4o] 페르소나 System 프롬프트 + 데이터 User 프롬프트
    │   스트리밍 응답
    ▼
프론트엔드에서 실시간 텍스트 렌더링
```

|------|-----------|----------------------|
| 사주 분석 (standard) | 1회/실행 | ~2,500 토큰 |
| 오늘의 운세 (daily) | 1회/실행 | ~1,800 토큰 |

| 꿈해몽 | 1회/실행 | ~2,200 토큰 |

> gpt-4o 기준 (2026년 3월): 입력 $2.50/1M tokens, 출력 $10.00/1M tokens
