## 1. 서비스 개요

**운세 에이전트**는 사주팔자·타로 카드·꿈해몽 세 가지 운세 서비스를 통합 제공하는 AI 기반 웹 애플리케이션입니다.
각 서비스는 독립 모듈로 구성되어 있으며, 공통 게임화 시스템(스트릭·업적·도감)을 공유합니다.

### 핵심 설계 원칙
- **AI-First:** 해석은 LLM 프롬프트 엔지니어링으로 처리, 하드코딩 최소화
- **Modular:** 사주·타로·꿈해몽 로직은 서로 독립적인 모듈
- **Modern Spiritual:** 신비롭고 어두운 배경의 'Celestial Editorial' 디자인

## 3. 사주 분석

### 입력 (`/input`)

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| 이름 | text | ✅ | 분석 대상 이름 |
| 성별 | 남/여 toggle | ✅ | 남성/여성 선택 |
| 생년 | select (100년) | ✅ | 출생 연도 |
| 생월 | select (1~12) | ✅ | 출생 월 |
| 생일 | select (1~31) | ✅ | 출생 일 |
| 태어난 시간 | select (12지 시) | - | 자시~해시 12개 선택지 |
| 시간 모름 | checkbox | - | 체크 시 낮 12시(오시)로 계산 |

**모드 구분:**
- `mode=standard`: 풀 사주 분석 (6섹션)
- `mode=daily`: 오늘의 운세 (5섹션)

### 계산 로직 (`src/lib/saju.ts`)

**의존 라이브러리:** `lunar-typescript` (음력 변환)

1. 양력 생년월일을 음력으로 변환
2. 60갑자(천간 10 × 지지 12) 기반으로 년주·월주·일주·시주 산출
3. 각 기둥의 천간·지지를 한자·한글·오행으로 변환
4. 오행(목·화·토·금·수) 분포 집계
5. 동물 띠 산출

**반환 타입 `SajuResult`:**
```typescript
{
  year: { stemKor, branchKor, stemHanja, branchHanja, stemElement, branchElement }
  month: { ... }
  day:   { ... }
  hour:  { ... }
  animal: string       // 띠
  elements: Record<'목'|'화'|'토'|'금'|'수', number>
  dayMaster: string    // 일간 한자
  sajuYear: string     // 태어난 년도 간지
  unknownTime: boolean
}
```

### 결과 화면 (`/result`)

| 컴포넌트 | 설명 |
|----------|------|
| `PillarCard` × 4 | 년·월·일·시주 각각 천간+지지+오행 표시 |
| `ElementRadarChart` | 오행 5각형 레이더 차트 (진입 시 0→100% 애니메이션) |
| `ElementBar` | 오행별 가로 바 그래프 |
| AI 스트리밍 텍스트 | gpt-4o 박진인 페르소나 해석 실시간 출력 |

## 4. 타로 리딩

### 흐름 (단일 페이지 5단계)

```
input → shuffling → spread → cards → result
```

| 단계 | 화면 | 동작 |
|------|------|------|
| `input` | 질문 입력 + 직감 기호 선택 | 질문 텍스트 입력 |
| `shuffling` | 덱 셔플 애니메이션 | 덱 3레이어 애니메이션, 직감 선택(별/달/태양) |
| `spread` | 7장 펼쳐진 카드 | 3장 선택 (순서대로 상황/조언/결과) |
| `cards` | 선택한 3장 플립 | 0.8s 간격으로 순차 뒤집기 |
| `result` | AI 해석 스트리밍 | gpt-4o 마담 셀레스트 해석 출력 |

### 카드 데이터 (`src/lib/tarot.ts`)

- **전체 카드:** 78장
  - 메이저 아르카나: 22장 (0번 바보 ~ 21번 세계)
  - 마이너 아르카나: 56장 (완드/컵/소드/펜타클 각 14장)
- **카드 속성:** `id`, `name`(영문), `nameKor`(한글), `suit`, `symbol`, `keywords[]`
- **`drawCards(n)`:** Fisher-Yates 셔플로 중복 없이 n장 추출
- **`formatCardsForPrompt(cards)`:** 카드 정보를 프롬프트용 문자열로 포맷

### 역방향(역위) 처리

카드 선택 확정 시 각 카드에 `Math.random() < 0.25` (25% 확률)로 역방향 부여.
역방향 카드는 UI에 `역` 배지 표시 + AI 해석에 반영.

### 카드 도감

`localStorage`에 확인한 카드 ID 목록을 누적 저장.
타로 페이지 헤더에서 전체 78장 중 본 카드 수 표시.

## 5. 꿈해몽

### 입력 항목

| 항목 | 타입 | 설명 |
|------|------|------|
| 꿈 내용 | textarea (6행) | 자유 형식 꿈 서술 |
| 감정 | 복수 선택 (8종) | 기쁨/불안/두려움/혼란/평온/슬픔/설렘/당혹 |
| 테마 | 복수 선택 (12종) | 비행/추락/물/불/동물/사람/죽음/탈출/집/길/이성/직장 |

**감정 선택 인터랙션:** 선택한 감정에 따라 아라의 반응 문구 + 글로우 색상 실시간 변경

### 꿈 유형 자동 분류

AI 응답 첫 줄의 `__TYPE__[유형]__` 태그를 파싱하여 배지 표시.

| 유형 | 색상 | 의미 |
|------|------|------|
| 길몽 吉夢 | 앰버 | 행운·좋은 일 예시 |
| 예지몽 豫知夢 | 보라 | 미래를 미리 보는 꿈 |
| 정화몽 淨化夢 | 블루 | 내면 정화·치유 |
| 불안몽 不安夢 | 오렌지 | 현재 스트레스 반영 |
| 흉몽 凶夢 | 레드 | 경고의 꿈 |
| 일상몽 日常夢 | 그레이 | 일상 기억 반영 |

### 꿈 일기

- 해석 완료 시 자동 저장 (`localStorage`, 최대 30개)
- 저장 데이터: 날짜·꿈 내용 120자·꿈 유형·감정·테마·AI 요약 100자
- 헤더 "꿈 일기" 버튼으로 모달 조회
- 개별 항목 삭제 가능

## 6. 게임화 시스템

담당 모듈: `src/lib/gamification.ts` (localStorage 기반)

### 스트릭 (연속 방문)

- 하루 1회 이상 운세를 본 날을 기준으로 연속 일수 집계
- 어제 방문 + 오늘 방문 = 스트릭 유지
- 어제 미방문 시 스트릭 초기화 (오늘부터 재시작)
- 홈 화면에 🔥 배지로 표시

### 데일리 도장 (스탬프)

- 오늘 사용한 서비스를 도장으로 기록
- 사주(仙·보라) / 타로(✦·골드) / 꿈해몽(☽·블루) 각 도장
- 3종 모두 찍으면 "✦ 오늘 삼합 달성!" 표시
- 월별 캘린더에 도장 기록 시각화

### 업적 (Achievement)

| ID | 이름 | 해제 조건 |
|----|------|-----------|
| `first_saju` | 첫 사주 분석 | 사주 분석 최초 완료 |
| `first_tarot` | 첫 타로 리딩 | 타로 해석 최초 완료 |
| `first_dream` | 첫 꿈 해몽 | 꿈해몽 최초 완료 |
| `all_three` | 삼합(三合) 달성 | 세 서비스 모두 최초 완료 |

- 업적 해제 시 `AchievementToast` 컴포넌트로 우상단 팝업 알림
- 홈 화면 하단 업적 섹션에 배지 표시

| `MongshinAra.tsx` | 몽신 아라 夢神 | 꿈해몽 | 블루 `#60A5FA` |

### SVG 캐릭터 사양
- viewBox: `200 × 300` (2:3 비율)
- prop `variant`: `'card'` (홈 카드, 120×180px) / `'hero'` (기본, 추가 장식 포함)
- prop `imagePath`: 실제 이미지 경로 전달 시 SVG 대신 이미지 렌더링 (확장 가능)
- 모든 그라디언트·필터·애니메이션 인라인 SVG로 처리 (외부 의존 없음)

### 캐릭터 표시 위치
- **홈 카드:** 각 서비스 카드 상단에 120×180px 포트레이트 + 하단 페이드 오버레이
- **사주 결과 로딩:** 仙 한자 글로우 아바타 + 로딩 메시지
- **타로 입력:** ✦ 글로우 아바타
- **꿈해몽 입력:** ☽ 글로우 아바타

## 9. 데이터 저장 방식

모든 사용자 데이터는 **브라우저 localStorage에만 저장**됩니다.
서버·DB 저장 없음. 브라우저 데이터 초기화 시 모두 삭제됩니다.

| localStorage Key | 내용 | 형식 |
|-----------------|------|------|
| `fortune_stamps` | 날짜별 사용 서비스 기록 | `{ "2026-03-31": ["saju","tarot"] }` |
| `fortune_achievements` | 해제된 업적 ID 목록 | `["first_saju", "all_three"]` |
| `tarot_seen_cards` | 타로 도감 확인 카드 ID | `["major_0", "cups_1", ...]` |
| `dream_diary` | 꿈 일기 항목 (최대 30개) | `DreamEntry[]` |
